# CHUSO1800 Lua Core 仕様書

> 対象: `src/chuso1800_core.lua`、`deploy/main.lua`、`main.sw-net`  
> 原型: [`../CHUSO1800_Traction_Controller/SPEC.md`](../CHUSO1800_Traction_Controller/SPEC.md)  
> 信号の正確な割付: [`SIGNAL_MAP.md`](./SIGNAL_MAP.md)

## 1. 位置づけ

Lua Coreは、1800系牽引制御マイコンのうち、直列・並列・界磁制御の状態機械、
カム進段、電流モデル、回生・空気ブレーキ補完を1つのLuaノードへ統合した実装である。
パンタグラフ、SAP/ECB変換、方向合成、Momelink-AおよびRolling Stock Statusの
フレーム整形は`main.sw-net`側に残す。

本書は**現在のLua Coreの挙動**を定義する。原型ゲート網との差分の理由は
`DESIGN_LOG.md`、配列スロットとビット位置は`SIGNAL_MAP.md`を参照する。

内部名には旧称が残る。

| Lua内部名 | 意味 |
|---|---|
| `phase1_latch` | 直列接続 |
| `phase2_latch` | 並列接続 |
| `regen_latch` | 界磁制御。力行と回生の両方で使用 |
| `eb_condition` | 非常制動に限らない牽引禁止条件 |
| `bc_target_smooth` | Momelink-Aへ送る自車平滑加速度 |
| `bcT` | 空気ブレーキ補完減速度要求 |

## 2. 実行モデルと契約

```lua
stateless_out, state_out = core_tick(stateless_in, state_in)
```

4配列はいずれも要素数8のLua数値配列である。`state_out`を次tickの
`state_in`へ戻す。永続グローバルには制御状態を持たない。

- ラッチ、タイマー、電流モデルの反復値、平滑化値は`state_in`の旧値で判断し、
  新値を`state_out`へ書く。
- その他の組合せ論理は1回の`core_tick`内で確定する。
- 原型ゲート網の「ゲート1段につき1tick」を逐語再現しないため、過渡のtick数は
  短縮されうる。定常状態と安全上の出力条件を互換性の基準とする。
- `deploy/main.lua`は`state_sync.lua`との境界で、stateスロット3～7だけを
  float32のビットパターンと整数の間で変換する。

## 3. 入出力

### 3.1 ステートレス入力

| slot | 値 |
|---:|---|
| 1 | 車両速度 [m/s] |
| 2 | ゲート側で選択済みの牽引供給電圧 [V] |
| 3 | 牽引禁止判定用ブレーキ管相当圧 [atm] |
| 4 | SAP/ECB解決済みのブレーキ要求圧 [atm] |
| 5 | 方向符号 `-1 / 0 / +1` |
| 6 | 力行ノッチ。整数化後0～7へ制限 |
| 7 | Controller Stop。0以外でtrue |
| 8 | DB自動。0以外でtrue |

### 3.2 ステートレス出力

| slot | 値 |
|---:|---|
| 1 | 電機子電流 [A] |
| 2 | 主回路電力 `W` |
| 3 | 自車平滑加速度。Momelink-A N2/N26へ送出 |
| 4 | 空気ブレーキ補完減速度要求 |
| 5 | 状態ビットフィールド |
| 6～8 | 予備。0 |

stateスロットとslot 5のビット割付は`SIGNAL_MAP.md`を正典とする。

## 4. ゲート側で解決する入力

Lua Coreは次の判定を行わない。

- SAP車かECB車かの選択と圧力換算
- 前進・後進booleanから方向符号への合成
- 外部架線電圧と定格1500 Vの選択、パンタグラフ有効判定
- 1800形・1900形のMomelink-Aフレーム選択

Lua Coreには、これらを解決した最終値を渡す。

## 5. 牽引禁止

次のいずれかで`eb_condition`を成立させる。

- Controller Stop
- 方向符号が0
- `abs(speed) > property.getNumber("Over Speed Th. [m/s]")`
- ブレーキ管相当圧が4 atm未満

成立中は有効ノッチを0とし、電機子電流、電力、電気加速度、界磁電流を0へ
置換する。空気ブレーキ補完要求は全量フォールバック値を使用する。

## 6. ノッチとカム軸

- 力行ノッチは0～7。
- カム位置は0～20の21段リングで、周期パルスごとに`(position + 1) % 21`。
- `cam_pulse`は位置が変化したtickでtrue。通常の`+1`と`20 -> 0`を含む。
- 直列域は0～13、直並列転換位置は14、並列域は14～20。
- 直列・並列がともにOFFでカムが0以外なら、0へ戻るまでカムを進める。

カム進段パルスは有効化から12tickごとに1回発火する。原型の
BLINKER+PULSEと定常周期は同じだが、初回位相は逐語互換ではない。

## 7. 直列・並列・界磁制御

### 7.1 状態

| 内部状態 | 主回路上の意味 |
|---|---|
| `phase1_latch` | 直列接続 |
| `phase2_latch` | 並列接続 |
| `regen_latch` | 界磁制御 |

### 7.2 通常シーケンス

1. ノッチ投入、カム0、並列OFFで直列をセットする。
2. ノッチ2以上、直列成立と電流制限判定が各6tick継続したら直列域を進段する。
3. ノッチ3以上、カム14、電流制限判定成立で並列をセットし直列をリセットする。
4. 並列成立と電流制限判定が各6tick継続したら並列域を進段する。
5. 並列状態でカム0へ戻ると界磁制御をセットする。

### 7.3 限流進段

進段電流しきい値は直列で`Power Limit Current [A]`、並列で同値-20 A。
しきい値未満が6tick連続した場合に進段可能とする。

### 7.4 カム復帰

直列・並列がともにOFFでカムが0以外の場合、カムを進めて0へ戻す。

### 7.5 界磁電流超過

ノッチOFF中の界磁電流しきい値は通常300 A、直列かつ回生減速度要求中は
400 A。条件が30tick継続するたびにパルスを発生する。

- DB自動ON: 並列から直列へ降格する。
- DB自動OFF: 直列・並列を解放する。

### 7.6 固着解除

次をすべて満たす場合、並列を直接解放する。

- 並列と界磁制御がON、直列がOFF、カム0
- ノッチ・回生制動要求がなく、電機子電流が±50 A以内
- `abs(speed) < 3 m/s`

この条件は界磁電流超過パルスへ混ぜず、直列セット経路を通さない。

### 7.7 強制全解放

次の場合は直列・並列・界磁制御を全解放する。

- `eb_condition`成立
- DB自動OFF中に直列と界磁制御が同時にON

解放を決めたtickは、旧ラッチ状態で物理演算済みであっても、電機子電流と
電力を0へ上書きする。Momelink-Aへ送る平滑加速度も平滑化状態ごと0にする。

## 8. 電動機モデル

主要定数は原型`n409.lua`に由来する。

| 定数 | 値 |
|---|---:|
| `K` | 12.16 |
| `Kmu` | 0.00029 |
| `MOT_RES` | 0.07 Ω |
| `Ks` | 0.85 |
| `PHIs` | 150 |
| `MOT_CTRL` | 4 |
| `GEAR_RATIO` | 5.31 |
| `WHEEL_R` | 0.43 m |
| `WEIGHT` | 35000 kg |

回転数、磁束、電機子電流は次式を基礎とする。

```text
rpm = speed * 9.55 * GEAR_RATIO / WHEEL_R
phi = iF * Kmu * Ks * PHIs / (Ks * abs(iF) + PHIs)
K * phi * rpm - terminal_voltage
  + (MOT_RES + external_resistance) * armature_current = 0
```

最後の方程式をNewton法で5回反復する。直列時は架線電圧と抵抗を1/8、
並列時は1/4として1電動機相当に換算する。

原型との差分は、界磁制御力行のノッチ4以上で、電機子電流目標を固定200 Aではなく
`Power Limit Current [A]`とする点である。ノッチ1～3は界磁電流追従を維持する。

## 9. ブレーキ要求

解決済みブレーキ要求圧から回生減速度目標を求める。

```text
regen_bc_target = -floor((brake_demand_pressure - 1) * 2) / 7.2
```

`regen_bc_target < -0.05`かつDB自動ONで回生制動要求を成立させる。
電気制動で不足する量を`bcT`として正値出力し、ゲート側で
`bcT * 3.6 + 1`によりBC絶対圧目標へ変換する。

## 10. 平滑化と回生抑制

- 回生減速度指令は、強める方向へ最大0.1 m/s²/tick、解除方向へ
  最大0.02 m/s²/tickで追従する。
- 自車加速度は`new = accel * 0.2 + old * 0.8`で平滑化する。
- 全解放時は後者のEMAを迂回し、出力と保存状態を即時0にする。

界磁電流保護の抑制タイマーは、0.5秒充電・10秒放電相当である。
内部では0～600の整数レベル（充電+20/tick、放電-1/tick）とactiveビットを持つ。
一度満充電でactiveになった後は、レベルが0へ戻るまでactiveを維持する。

## 11. 実装上の制約

- Stormworksの8192文字制限に収めるため、コアはモジュールテーブルを持たず、
  外部APIをグローバル関数として定義する。
- 内部の値受け渡しは位置引数・多値返却を使う。引数順は各関数直前のコメントと
  テストハーネスを正典とする。
- ビット演算は、storm-lua-minifyが括弧を誤って削除する既知問題を避けるため、
  異なる優先順位の演算子を1つの式へ混在させない。
- Newton反復は、minifierのローカル変数名衝突を避けるため`physics_tick`へ
  インライン化してある。

## 12. ゲート側に残す機能

- パンタグラフ4ラッチと1800/1900形判定
- 架線電圧選択
- SAP/ECB変換、方向合成
- Momelink-A Advanced/1800形フレーム整形
- Rolling Stock Status整形

Momelink-A AdvancedフレームはN26まで書き込むため、`main.sw-net`の
`momelink_advanced_frame`は`count=26`でなければならない。

## 13. 検証条件

```sh
lua test/run_all.lua
lua test/verify_deploy_artifact.lua
```

前者はソースの状態遷移・物理回帰・実運転シナリオを、後者はminify済み成果物を
直接実行し、ビット演算と変数名衝突の回帰を検出する。

安全上の不変条件は次のとおり。

- 直列・並列がともにOFFなら、電機子電流、電力、自車平滑加速度は0。
- 非常制動または強制全解放を決めたtickから上記3出力は0。
- DB自動OFF中は直列＋界磁制御状態を維持しない。
- 低速固着解除は直列を誤セットせず、並列だけを解放する。

## 14. 保守方針

- 現在挙動は本書、信号割付は`SIGNAL_MAP.md`、決定理由は`DESIGN_LOG.md`へ置く。
- ソースコメントには、その行を安全に変更するために必要な局所条件だけを書く。
- 原型ゲート網との逐語一致より、明示した差分と安全上の不変条件を優先する。
- `regen`という語は、DB自動、界磁制御、回生制動要求、回生抑制を区別して読む。
